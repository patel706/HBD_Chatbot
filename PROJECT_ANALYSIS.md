# Project Analysis — HBD Local Business AI (botpro)

Summary

- **Type:** Full-stack bot project (FastAPI backend + Streamlit admin UI + React/Vite frontend)
- **Purpose:** Provide a local-business assistant that searches a SQLite DB, can search online, and lets business owners manage profiles via chat/OTP.

How to start

- Backend (Windows, from project root):

```powershell
cd backend
REM create & activate a venv if missing
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
REM start (uses uvicorn to serve api:app on 127.0.0.1:5000)
.\.venv\Scripts\python.exe -m uvicorn api:app --host 127.0.0.1 --port 5000 --reload
```

- Frontend (root):

```bash
npm install
npm run dev
```

- Streamlit UI (optional admin/chat UI):

```powershell
cd backend
REM activate .venv then run
streamlit run app.py
```

Files and flow (high level)

- `backend/START_SERVER.bat` — installs requirements into `.venv` and runs `uvicorn api:app`.
- `backend/api.py` — FastAPI app, mounts static files, provides endpoints for OTP, search, uploads, and business CRUD.
- `backend/app.py` — Streamlit-based chat/admin UI that calls local DB and some backend functions.
- `backend/*.py` (helpers) — business logic: `business_by_phone.py`, `assistant_manager.py`, `llm_client.py`, `text_to_sql.py`, `search_online.py`, etc.
- `backend/google_map_data.db` — SQLite DB file (created/used by scripts) — DB path resolved in `api.py` and `START_SERVER.bat`.
- `public/` and `src/` — React frontend (Vite) components and pages that call the API.

Common issues I found

- Project expects a local Python virtual environment at `backend/.venv`. If missing, `START_SERVER.bat` will fail to run Python commands.
- `backend/requirements.txt` includes `streamlit` and `fastapi` etc.; ensure you're in the `.venv` when installing.
- `api.py` prints DB path and assumes `google_map_data.db` exists; if it does not, some endpoints will error. Create or provide that DB or run migration scripts (`migrate_data.py`).
- CORS: `FRONTEND_ORIGIN` defaults to `http://localhost:3000`; your Vite dev server uses port 5173 — `api.py` already allows 5173 but consider setting `FRONTEND_ORIGIN` in a `.env` file.
- Email OTP: `send_smtp_otp` requires `SMTP_EMAIL` and `SMTP_PASSWORD` in `.env` to send real emails; otherwise opt for phone OTP (mock code `1234`).
